Why · 01

기획부터 배포까지 나흘. 사람은 방향을 정하고, 코드는 전부 AI가 썼습니다. 그 결과물이 D-Garden입니다.

How · 01

점수·가중치·색상까지 전부 명세 파일로 — 기획자가 코드 없이 직접 튜닝

✅

TDD — 자동 테스트 49개

코드 대신 GREEN/RED 신호등을 봅니다 — 로직이 깨지면 즉시 발각

How · 02

나흘의 기록,
커밋 38개

DAY 1 — 6/9 v1.0 탄생 멘탈 스코어 엔진(TDD) + 프랙탈 나무 + 물리 바람 시뮬레이션
DAY 3 — 6/11 세계관 확장 + 세상에 공개 4종 테마 · 전역 입력 에이전트(브라우저 밖 타이핑도 감지) · Supabase 연동 · Vercel 배포
DAY 4 — 6/12 무대 준비 관객 6버튼 리모컨 · Zoom식 반응 버블 · 데모용 점수 튜닝

How · 03

가장 화려한 화면을 만든 건
가장 단순한 기술이었습니다.

0 프레임워크
(Vanilla JS)
0 LLM API 호출
(경량 알고리즘)
49 자동 테스트
(품질 신호등)

스프링 물리, 3중 주파수 바람, 프랙탈 생장 — 전부 순수 웹 기술입니다. 비개발자 혼자서는 불가능했지만, AI와 함께라면 나흘이면 충분했습니다.
